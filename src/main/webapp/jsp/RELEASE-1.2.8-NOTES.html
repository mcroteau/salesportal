Sales Portal 1.2.8 (Feb 2009)

Sales Portal 1.2.8 has a number of new features and some fixes that are aimed at helping you to grow your business

New Features highlights

  • Maintenance over Data Migration options and Data Migration Tables added
  • Prospect Import changes for territory and status id - see below
  • Totally new Logic for outbound Emails - see below
  • Import - added check box to check for duplicates, plus added to appConfig.xml for automated imports
  • New Territories to default to Yes include in Information Portal reports
  • Prospect Last Change date - now updated if notes, contact, sales action or campaign is added or updated
  • New Search criteria added to work with prospects screen - see below
  • Fixed a bug on return to Calendar from prospect detail screen


      • Below is a more detailed list of changes and updates to Sales Portal for this release

      Outbound Emails to Prospects

      • Sales Portal can now easily send outbound emails to prospects via interfaces to Order Portal
      • Here is a link to the documentation on the web site that gives detailed information )Email Demo Script
      • Here is a summary of the changes
        1. In Order Portal - You can now manually delete/remove an entry in the Email subscribers table
        2. In Order Portal - Draft Emails are now linked to a webprofile. They use the from email and smtp settings from that profile
        3. In Order Portal - Draft Emails send to any email in the Email Queue table (a new table)
        4. In Order Portal - You can select from the Email Subsriptions and the customer table to build your list in the Email Queue
        5. In Order Portal - You can now manually import a list of email addresses to send Emails to
        6. When you create a CSV file from the work with prospects screen you can now manually import it into Order Portal for sending of Emails
        7. There are currently 2 forms of automated email interface
          1. From work with prospects screen there is a new option to send Emails
          2. In admin there is a new Email Sales Actions that allows you to define a prospect group, date and email to be sent
        8. There is an automated interface between Sales Portal and Order Portal for importing prospects into the Email Queue and automatically sending Emails
          1. This is part of the base Web Business Suite logic. Web Business Suite consists of:
            1. Sales Portal
            2. Order Portal
            3. Information Portal for Order Portal and Information Portal for Sales Portal
            4. Data Migration Portal
            5. Forum. (We normally use jForum)
          2. See the link above for complete documentation. There will also be screen shots on the webs sites shortly
          3. Automated Email Links requires the latest Sales Portal, Order Portal and Data Migration Portal

      Import Changes

      • Additional help text has been added to the import screen
      • There is an added option that determines whether Sales Portal checks for duplicates as it imports or creates a prospect for every record
      • Territory - you can now use territory id as well territory name
      • Status code - you can now use status id as well status name

      Data Migration Portal

      • The data base scripts for Data Migration Portal are now included in the Sales Portal database
      • the Data Migration Portal options for prospect importing are now available on the Configuration menu under Data Migration Options

      Prospect Search Changes (work with prospects screen)

      • You can search on email
      • You can search on phone number
      • You can search against contacts by name or phone number
      • You can search on prospect last change date, either exact match or from this date forware
      • Prospect change date is updated when:
        1. Add or update a Prospect Note
        2. Add or update a Prospect Contact
        3. Add or update a Prospect Sales Action
        4. Add or update a Prospect Campaign
